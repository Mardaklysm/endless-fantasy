# Endless Validation Matrix

Pick the smallest validation that covers the change. `npm run build` is the default code validation, and `git diff --check` should run before commit.

## docs-only

Use for: `agent_knowledge/`, README, comments, prompts, or non-runtime docs.

Validation commands:

```powershell
git diff --check
```

What not to run by default: `npm run build`, `npm test`, Vite, browser smoke.

## code-static

Use for: TypeScript-only edits, data tables, settings models, non-visual logic with no worldgen semantic changes.

Validation commands:

```powershell
npm run build
git diff --check
```

What not to run by default: `npm test`, Vite, browser smoke.

### Visual / Worldgen Renderer Tasks

Use for:
- semantic terrain renderer changes
- biome material blending
- terrain mask visual regressions
- worldgen visual previews
- screenshot-seed terrain issues

Validation:
- `npm test`
- `npm run build`
- `git diff --check`

Optional:
- `npm run worldgen:lab -- --seed <seed> --out tmp/worldgen-lab/<name>` only when the task includes a specific seed, screenshot, or visual regression.

Do not run by default:
- Vite/dev server
- browser automation
- temporary runtime smoke pages

Browser smoke limit:
- Browser smoke gets one attempt only.
- If there is no supported seed/debug entry path, stop and report the blocker.
- Do not keep trying localStorage injection, page-global mutation, or `javascript:` URL workarounds.
- Do not create temporary smoke harnesses/pages unless the user explicitly asks for visual proof or the task cannot be verified any other way.

What not to run by default: browser smoke unless visual/runtime behavior also changed.

## browser-smoke

Use for: visual/runtime behavior that build cannot verify, or when the user explicitly asks.

Validation commands:

```powershell
npm run dev
```

Then inspect the changed surface at `http://127.0.0.1:5173`.

What not to run by default: for docs, settings/data-only changes, or static code changes.

## full-regression

Use for: broad cross-system work, release checks, or explicitly requested regression passes.

Validation commands:

```powershell
npm test
npm run build
git diff --check
```

Add browser/manual checks only with a concrete reason.

What not to run by default: full regression should be rare and explicitly justified.
